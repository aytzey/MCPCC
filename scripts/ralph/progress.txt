## Codebase Patterns
- Rust workspace lives at repo root (`Cargo.toml`) with member crates under `crates/`.
- Run `cargo fmt --all --check && cargo test && cargo check` before committing.
- On this runner, `/projects` can fill up; set `CARGO_TARGET_DIR` to a path on `/tmp` for builds/tests and use `cargo clean` if you hit ENOSPC.
- For integration tests, run binaries via `env!("CARGO_BIN_EXE_<bin>")` instead of shelling out to `cargo run`.
- For compiler-invocation tests, point `--mcpcc-cc` at a temp-dir shell-script "fake compiler" to assert argv + exit-code behavior.
- When integration tests invoke `mcpcc`, set `Command::current_dir` to a temp dir so generated artifacts don't pollute the repo.
- Use `mcpcc::plan_artifacts()` to detect link vs compile-only and compute deterministic artifact paths (with wrapper overrides).
- MCP generation errors after a successful compilation must exit with code 70.
- Use atomic JSON writes (temp file + parse validation + rename) for both `*.mcp.json` and `*.mcpcc-manifest.json`.
- LLM is `required` by default; tests can use `--mcpcc-llm-mode=best-effort` (or `off` with `MCPCC_ALLOW_NO_LLM=1`) to avoid OpenRouter requirements.
- For LLM integration tests, point `MCPCC_OPENROUTER_BASE_URL` at a local mock server and set `--mcpcc-cache-dir` to a temp dir to avoid global cache/network.
- Structured tool extraction is best-effort; failures should be recorded in manifest `analysis.notes` and only printed under `--mcpcc-verbose`.
- Tool base names are normalized to `[a-zA-Z0-9._-]` and truncated to 120 chars so `<base>.run_raw` stays within 128 characters.
- `mcp.json` always includes `tool.x-mcpcc.exec` defaults (`timeoutMs=30000`, `maxStdoutBytes=maxStderrBytes=1048576`); tool annotations can override.
- Extractor preference is: annotations (merge) > `argp` > `getopt_long` > fallback-only (`<base>.run_raw`).
- Structured tools embed argv mapping metadata in `tool.x-mcpcc.argvMapping` with deterministic `options[]` ordering and an `argsParam` for positionals.
- When an extractor provides per-option docs, populate `inputSchema.properties.<param>.description` for better tool-call UX.
- In `mcpcc_annot.h`, use a `MCPCC_CONCAT()` helper so `__COUNTER__` expands when token-pasting; otherwise GCC may treat it literally and cause symbol redefinition.
- When annotation tests use a real system compiler, pre-create output directories (e.g. `bin/`) since fake compilers `mkdir -p` but `cc`/`ld` won't.
